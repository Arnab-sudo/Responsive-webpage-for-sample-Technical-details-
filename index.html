What is HTML

HTML stands for HyperText Markup Language. it is the foundational script that is used to build a webpage. though it is not technically a programming language, it is qute extensive and is quite useful in building a basic static webpage. This is based on a set of principle that has been agreed on by international stahdards to regulate and have a uniform structure for the internet. But the HTML codes dont work on anywhere other than that of a Web browser which isoften written in other low level languages like C++. the browser interprets the HTML code and executes them to create the webpage.

As we can see that the tags like html, body, enclosed withinb the conditional operators are actually HTML tags that builds the herirarchy structure of the webpage. there rae many more such HTML tags and elements, each having attributes like width and height that helps us build the web document.

What is CSS

CSS stands for Cascading Style Sheets. this isused by the HTML document to add some styling to the basic html document, which makes it lookpresentable to the visitors.
it also adds behaviour to the html document and makes it responsive to user input and viewport dimensions. it also adds animations to the elements. the syntax of css file looks like the following: 

                
                  #text{
                    color: black;
                  }

here we see that the '#text' is the selector of the 'id' attribute of am element , and that is being used to set the color of the content as black. this example was that of a single selector, while there are many more types, like combined selectors, chained selectors,etc.

Static vs Dynamic HTML

  • The easiest difference is static HTML once rendered cannot be changed on the other hand dynamic HTML can be changed.
  • Static web pages cannot have database access but dynamic pages can be connected to database.
  • Using static HTML no interactions persist between user and server but dynamic HTML has capability to interact with the user.
  • Static HTML does not require server side coding but dynamic HTML requires server side coding.
  • No animation, games can be developed using the static HTML but on the other hand dynamic HTML can perform this task.

var vs let vs const

Var is the classical way of js coding...ot is hoisted and can be redecorated and updated. It is limited to the the scope it's is defined in is global scope , func scope etc. Variables declared using var that are global can be accessed by functions which are also capable of declaring their own variables with same name but diff values....this creates an error.


This is where the let variables come in...let variables are also hoisted. They are block scoped that means they are only relevant in the blocks they have been defined meaning that any let variable outside a function is not global scoped. In a scope they can be updated but not redeclared. But in different scopes let variables can share same name as they won't be relevant outside of it.


Const is similar to let only that it can't be updated either since it's constant.

Web Applications

normally what happens is that the client demands a certin resource or data from the server and the web sever delivers it by fetching them from the database. but this is a very lengthy process and eats lot of time and CPU power of the server. thus was introduced the concept of 'web-applications'.


normally when the resources were demanded from the server, all the sttaic(html, css) files were fetched and stored in web server n every instance. but now what web application sdo is that before the server can respond to the HTTP GET request of the client, it takes the data, specific to the request and adds them to pre-builthtl templates that were stored as static(bcpoz they dont change)files in the static folder. thus, as soon as the server receives a GET/POST request from the server, it sends the request data, aong with its url encoding(which page, sub-page,etc), the GET/POST data , its headers and the cookies to the web appplication. now, the web app, is fed the HTML templates that is same for a certain typeof GET/POST request, from the static files. a copy of the static file is also sent to the server to minimise the instances of server calls so that same resources can be stored and reduce downtime server. now the web application gets the data to fill the html template from the database, like mongodb, mySQL,etc. if its a GET request, the web app will withdraw data but in an event of POST request the web app might be used to add or updatethe databse data. in the end, the web app, adds the html template, fills it with the data from database and then sends the modified html, with the styling, data, and bells and whistles, to server. and now the server havnig recieved the finsihed response, trasnmits that to the browser/app thereby reducing the browser workload, incerasing speed, and performance.

Frontend vs Backend

Frontend

Fronted refers to the presentational logic ofa document, that is the webpage. we use the html, css and javascript. we use that to make a vanilla webpage.
nowadays we use the frontend libraries, APIs, to create webapplications and use the frontend librbaries to build dynamic webpages, that are responsive.
This script is what gets passed onto and is procesed by the browser. initially it dies reside in the server as well, but on first server request, it gets downloaded and resides in the host computer to reduce downtime as they seldom change.


Backend

Backend represents the business logic of the website. the backend codes were mostly written in php and these codes reside in the server computers that are located in the data-firms. but now a days, we can use varous other languages that can be used to write backend codes, namely, C#, Java, Python and also Javascript.
what the backend code does is that it defines the logic as t ohow the severs shud respond to specific requests. there are two types of requestsm GET and POST requests. depending on them, the server responds. the response is defined by the functions that are written by the user. normally this requests are to extract the field values and data from the database.

Databases

Databases contain the data that the servers read from. the databases are today of various types. the famous one is ofcourse the SQL database that uses the SQL langauge to query the data base of the needed data. the SQL is a relational database, which organises the data in tables and rows to give it a structure. however the tabular formatting also means it needs more memory and is not as scalable.
this is where the NoSQL databses as used by the MongoDB and Django is useful as they oranise the data into objects. this makes them mroe scalable, easy to add and delete fields. also this means that we can orgaises the complexdata in to JSON objects which occupies lesser memory. and just in case we seek to access a tabular format of these data, we can easily orgainse tha data from them using the Django models, or by usng the Mongoose library written in Javascript.
